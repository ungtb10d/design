enum SubstanceAir::Renderer::RunOption

Run options enumeration used as argument by run() method Options can be combined. Run_PreserveRun is only useful w/ Run_Replace and/or Run_First flags (useful w/ burst run() calls).

Enumerator
Run_Default 

Synchronous, preserve previous push.

Run_Asynchronous 

Asynchronous computation.

Run_Replace 

Discard computation on deprecated outputs.

Run_First 

Run w/ maximum priority on other computation.

Run_PreserveRun 

In any case, preserve currently running job.

◆ push() [2/2]

void SubstanceAir::Renderer::push ( GraphInstance &  )

Push graph instance current changes to render The current GraphInstance state is used to enqueue a render job: input values and flags, output flags.

Only the outputs of the graph flagged as dirty (outdated) are rendered (use OutputInstance::flagAsDirty() to force rendering).
When a GraphInstance is pushed for the first time, all outputs are considered dirty (and therefore are computed later).

Note
Note that the rendering process is not started by this command but with the run() method.

◆ run()

UInt SubstanceAir::Renderer::run ( UInt  runOptions = Run_Default,
size_t  userData = 0 
)

Launch synchronous/asynchronous computation.

Parameters
runOptionsCombination of RunOption flags
userDataUser data transmitted to output complete callback. can be used to store pointer or other stuff.
Postcondition
If synchronous mode selected (default): Render results are available in output instances (OutputInstance::grabResult()).
Returns
Return UID of render job or 0 if not pushed computation to run.

Render previously pushed GraphInstance's. Does nothing if no computation are pushed (no previous call to push() or no dirty outputs in pushed instances).

Both synchronous/asynchronous render are processed on an internal render thread by default (useful for GPU Substance Engine versions). This behavior can be overridden by RenderCallbacks::runRenderProcess().

Returns after computation end or immediately if Run_Asynchronous mode is set and render is not processed on caller thread (using RenderCallbacks::runRenderProcess() callback).

If rendering is currently hold, a synchronous render call automatically resume it.

◆ setOptions()

void SubstanceAir::Renderer::setOptions ( const RenderOptions &  renderOptions)

Set new memory budget and CPU usage.

Parameters
renderOptionsNew render options to use.

The new memory budget and CPU usage are applied on current/pending jobs. This function is thread safe.

Note
If no computation is running, in order to apply the memory budget a call to run() is necessary (possibly without pushed computations). N.b. setOptions() is thread safe but run() is not.
